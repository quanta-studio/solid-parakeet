# Automation Script

## Strength of LLM-based automation
1. They understand natural language and automated the tasks based on the natural language input. This allows for a more intuitive and user-friendly interaction with the virtual engineers, as they can understand and execute tasks based on human-readable instructions.

## Downside of LLM-based automation
1. They are inconsistent. They can produce different automation results for the same input, which can lead to confusion and errors in the project. This is especially true when working with multiple virtual engineers, as they may produce different outputs for the same task.

2. Thus, in order to ensure consistency and reliability in the automation process, it's better to ask the virtual engineers to produce automation scripts that can be executed in a consistent manner. This way, the automation process can be standardized and the results can be reproduced reliably.

Prepare as much as automation scripts and turn them into either a command or agent skills that can be executed by the virtual engineers. This will help to ensure that the automation process is consistent. (Note: This is also a way to control the token consumption of the virtual engineers, as they will not need to process the same natural language input multiple times for the same task.)

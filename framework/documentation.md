# Documentation

## Breakdown

1. Breakdown the documentation into smaller and manageable parts.

2. Identify the topic that needs to be loaded into the virtual engineers' memory vs on-demand loading. This will help to optimize the memory usage of the virtual engineers and ensure that they have access to the necessary information when needed.

3. Utilise the SITEMAP.md and modular structure of the project to identify the relevant documentation for each module. This helps to ensure the virtual engineers have enough context and avoid overfeeding them with unnecessary information. (Which end up in costing more token consumption and context distraction.)

4. IF you are lazy, drop the documentation and get the virtual engineers to organize and breakdown the documentation in a way that is easy to understand and navigate. This can be done by creating a documentation index or table of contents that outlines the different sections and topics covered in the documentation. The virtual engineers can also create summaries or abstracts for each section to provide a quick overview of the content. (Though, this is gonna cost more token consumption, but it is a good way to ensure that the documentation is well-organized and easy to navigate for the virtual engineers. In a way, helps the virtual engineers to index and understand the codebase and documentation better, which is a good thing for the project in the long run.)
